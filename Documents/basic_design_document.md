
# モビルパワーズ App 基本仕様書

## 1. はじめに

本文書は、「モビルパワーズ App」の基本設計に関する仕様を定義するものです。
アプリケーションの目的、主要機能、対象ユーザー、および基本的なシステム構成について概説します。

## 2. アプリケーション概要

- **名称:** モビルパワーズ App
- **目的:** トレーディングカードゲーム「モビルパワーズ」のファンメイドサポートアプリケーション。ルール参照、カード閲覧、デッキ構築支援、およびCPU対戦機能を提供することで、プレイヤーのゲーム体験を向上させることを目指します。
- **対象ユーザー:** モビルパワーズのプレイヤー、および同ゲームに関心を持つユーザー。

## 3. 主要機能

1.  **ルールブック表示機能:**
    *   ゲームの公式ルール（またはそれに準拠する詳細なルール）を閲覧できる機能。
    *   セクションごとのナビゲーション、キーワード検索（将来的な拡張）。
2.  **カードビューワ機能:**
    *   全カードのデータベースを閲覧できる機能。
    *   カード名、種類、所属、ポイント、地形適性、タグ、テキスト等による絞り込みおよびソート機能。
    *   カード画像、詳細情報の表示。
3.  **デッキエディタ機能:**
    *   ユーザーが自由にデッキを構築・編集できる機能。
    *   デッキは55枚以上100枚以下の範囲で構築できます。
    *   カードの追加・削除、同一基本カード3枚までの制限管理。
    *   デッキコードの生成および読み込み機能によるデッキ共有のサポート。
4.  **CPU対戦機能:**
    *   ユーザーがCPUプレイヤーと対戦できる機能。
    *   デッキ選択（プリセットまたはカスタムコード）。
    *   ゲームルールの自動処理と進行。
    *   ルールベースAIによるCPUの行動決定。
5.  **メインメニュー:**
    *   各機能へのナビゲーションを提供する開始画面。

## 4. システム構成 (概要)

- **フロントエンド:** React, TypeScript, Tailwind CSS
- **状態管理:** React Hooks (useState, useContext, etc.)
- **カードデータ:** TSV形式で提供され、アプリケーション内で解析・利用。
- **CPUロジック:** `aiService.ts` 内にルールベースで実装。

## 5. UI/UX設計方針

- **直感的な操作性:** ユーザーが迷わずに各機能を利用できるよう、シンプルで分かりやすいインターフェースを目指します。
- **レスポンシブデザイン:** スマートフォン、タブレット、デスクトップなど、様々なデバイスで快適に利用できるよう対応します。
- **アクセシビリティ:** ARIA属性の適切な使用など、アクセシビリティにも配慮します。
- **視覚的魅力:** ゲームの世界観を損なわず、かつ現代的で見やすいデザインを追求します。

## 6. 今後の拡張可能性
