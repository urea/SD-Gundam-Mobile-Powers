# モビルパワーズ App 改修履歴

## 2025/05/28 (初期開発からの主な変更点)

- **手札エリアの高さ調整**:
  - プレイヤーの手札エリアがカードを2段表示できる十分な縦幅を確保するようにUIを調整しました。

- **レイアウトの高さ均等化**:
  - 中サイズ以上の画面で、プレイヤーエリアと「CPUエリア＋選択カード詳細」セクションが同じ高さになるようにレイアウトを調整しました。

- **選択カード詳細エリアのUI改善**:
  - カード画像がエリアの高さ全体を使って表示されるようにレイアウトを変更しました。
  - カード画像が利用可能なスペースに応じて段階的に大きくなるように調整しました（複数回）。
  - 中サイズ以上の画面では、カード画像が横幅の50%を使用するように大幅に拡大しました。
  - 小さい画面では、カード画像に情報が集約されているため、テキスト詳細を非表示にするように変更しました。

- **戦場共通情報エリアの高さ調整**:
  - 戦場共通情報エリア（ゲームログ、ステータス等を含む上部セクション）が、大きな画面で不必要に引き伸ばされず、コンテンツに応じた適切な高さになるように修正しました。

- **ゲームボード全体のレイアウト調整**:
  - 上部の共通情報エリアと、その下にあるプレイヤーエリアおよびCPUエリアが不必要に間延びせず、互いに近接して表示されるように調整しました。
  - 画面幅が非常に狭くなった場合でも、プレイヤーエリアとCPUエリアが縦に並ばず、常に横並びを維持するようにレイアウトを固定しました（コンテンツの画面外へのはみ出しを許容）。

- **選択カード画像のモーダル表示機能**:
  - 「選択カード詳細」に表示されているカード画像をタップまたはクリックすると、画面いっぱいにその画像を拡大表示するモーダル機能を追加しました。

- **戦場・小隊・手札カード表示のレスポンシブ対応 (小画面)**:
  - スマートフォン表示等の小さい画面で、戦場エリア・小隊エリア・**手札エリア**のカードが最低でも3枚横並びになるようにレイアウトを調整しました。
  - これに伴い、カード間の余白を削減し、カードサイズがコンテナ幅に応じて可変になるように変更しました。中サイズ以上の画面では既存のレイアウトを維持します。

- **全般**:
  - UI/UXの微調整とレスポンシブ対応の改善を継続的に実施。
  - ゲームロジックの段階的な実装とAI対戦機能の基礎開発。
  - カードビューワ、ルールページ、デッキエディタ（開発中）の各ページ機能の実装。
  - 捨て札モーダル、山札モーダルの追加。
  - ゲーム進行に応じたUIの視覚的フィードバック（フェイズ表示、CPU思考中インジケータ、戦闘結果アニメーションなど）を実装。

## (日付は最新の更新日を想定) データ構造更新と新ゲームメカニクス

- **データ構造の変更**:
  - カードデータにおいて、従来の「補足 (Other_Notes)」列を「**タグ (Tags)**」として扱うように変更しました。
  - Cカード（カウンターカード）専用の「**効果 (Effect)**」列を新たに追加し、Cカードの主要なルールテキストを格納するようにしました。
  - これに伴い、アプリケーション全体のカードデータ型定義 (`Card` interface) を更新しました。`textAbility`はMカードのフレーバー能力、Cカードのキャッチコピーを保持し、Cカードの実際の効果は新しい`effect`フィールドに格納されます。

- **TSVパーサー修正**:
  - カードデータローダーを修正し、従来のCSV形式ではなく、正しく**タブ区切り形式 (TSV)** のデータを解析するようにしました。

- **ルールブック更新**:
  - 「カードの種類と使い方」セクションで、「タグ」と「効果」に関する説明を更新しました。
  - 「戦闘について」セクションで、新しいタグベースの戦闘力ボーナスルールを追記しました。

- **カードビューワ改善**:
  - 「補足」列ヘッダーを「タグ」に変更し、カードのタグ情報を表示するようにしました。
  - 「テキスト」列は、Mカードの場合はフレーバー能力を、Cカードの場合は主要な効果テキストを表示するように変更しました。
  - 「タグ」列でのソート機能を追加しました。
  - タグによる絞り込みフィルターを追加しました。

- **ゲーム内表示の更新**:
  - ゲーム中の「選択カード詳細」パネルで、新しいデータ構造（タグ、効果）に基づいた情報が表示されるように更新しました。
  - カードのツールチップ情報も更新されました。

- **新ゲームメカニクス「タグボーナス」の実装**:
  - 戦闘計算フェイズにおいて、Mカードの戦闘力計算ロジックを更新。各Mカードは、自軍の他のMカード（同じく最前線フィールドにいるもの）のタグと自身のタグが一致する場合、その一致したタグの総数だけポイントが加算されるようになりました。

- **AIロジックの更新**:
  - CPUプレイヤーの思考プロンプトを更新し、新しい「タグ」情報とCカードの「効果」テキストを考慮するようにしました。

## コンボルールの追加
戦場にでたＭカードのタグを利用して判定。コンボにより加算されたことを画面上に表示。

| コンボ名称        | 成立条件                                                     | 加算ポイント |
|-----------------|--------------------------------------------------------------|----------|
| トリプルキラコンボ   | 同一ナンバーのキラカードが前線フィールドに 3枚並んだ時。                     | +10      |
| トリプルGコンボ    | 同一ナンバーの“ガンダム系”カードが前線フィールドに 3枚並んだ時。                | +8       |
| 大将軍コンボ       | “大将軍”カードが前線フィールドに 3枚並んだ時。                             | +8       |
| 闇コンボ          | “闇の支配者”カードが前線フィールドに 3枚並んだ時。                           | +8       |
| トリプルコンボ     | 同一ナンバーのカードが前線フィールドに 3枚並んだ時。                         | +7       |
| キラコンボ        | キラカードが前線フィールドに 3枚並んだ時。                                 | +5       |
| 機体系コンボ       | 同一機体系カードが前線フィールドに 3枚並んだ時。                             | +5       |
| パイロットコンボ    | “同一のパイロット”専用機カードが前線フィールドに 3枚並んだ時。                   | +5       |
